● # GitHub Issue 作成ウィザード                                 
                                                                
  ユーザーから作業内容を受け取り、プランモードで対応方針を固めて
  から GitHub Issue を作成する。                                
                                                                
  ## Step 1: 入力を受け取る                                     
                                               
  `$ARGUMENTS` が空でない場合はその内容を Issue 内容として使う。
  空の場合は現在の会話コンテキストから内容を把握する。          
                                               
  ---

  ## Step 2: 不明点のみ確認

  入力内容を解析してタイプを判定する:
  - 不具合・おかしな挙動 → **バグ修正**（label: bug, title
  prefix: `fix: `）
  - 新しい機能・画面 → **機能追加**（label: enhancement, title
  prefix: `feat: `）
  - コード整理・改善 → **リファクタリング**（label: refactoring,
   title prefix: `refactor: `）

  **必須情報が不足している場合のみ** AskUserQuestion
  でまとめて確認する。
  判明している情報は聞き直さない。

  バグ修正の必須情報: 再現手順 / 期待する動作 / 実際の動作
  機能追加の必須情報: 背景・目的 / 実装したい機能 / 完了条件
  リファクタリングの必須情報: 対象モジュール /
  現状の問題点と目標

  ---

  ## Step 3: Plan サブエージェントで対応方針を固める

  `Agent` ツールで `subagent_type: "planner"`
  のサブエージェントを起動し、以下を指示する:

  - 対象画面・モジュール・関連ファイルを調査して現状の実装を把握
  する
  - 変更箇所・手順・技術的な判断を明確にした実装方針を返す
  - **コードは変更しない**（調査・方針策定のみ）

  サブエージェントが返した方針を次の Step で Issue 本文の `##
  実装方針` セクションに含める。

  ---

  ## Step 4: Issue 本文を組み立てる

  収集した情報と策定した方針を Markdown で整形する。

  **バグ修正:**
  ```
  ## 対象画面
  {画面名}（{URL}）

  ## 再現手順
  1. ...

  ## 期待する動作
  ...

  ## 実際の動作
  ...

  ## エラーログ
  （あれば）

  ## 実装方針
  {Step 3 で策定した方針}

  ## 制約事項
  （あれば）
  ```

  **機能追加:**
  ```
  ## 対象画面 / モジュール
  {画面名}（{URL}）

  ## 背景・目的
  ...

  ## 実装したい機能
  ...

  ## 完了条件
  - [ ] ...

  ## 実装方針
  {Step 3 で策定した方針}

  ## 技術的なスコープ
  - [ ] DBスキーマ変更あり（マイグレーション必要）
  - [ ] APIエンドポイント追加・変更あり
  - [ ] 新規コンポーネント作成
  - [ ] 既存コンポーネントの変更

  ## 参考にすべき既存実装
  （あれば）

  ## その他の技術的な注意点
  （あれば）
  ```

  **リファクタリング:**
  ```
  ## 対象モジュール / URL
  {モジュール名}（{URL}）

  ## 目的
  ...

  ## 現状 → 目標
  ...

  ## 実装方針
  {Step 3 で策定した方針}

  ## 変えてはいけないこと
  （あれば）

  ## その他の制約・補足
  （あれば）
  ```

  ---

  ## Step 5: Issue を作成する

  本文の末尾に必ず以下の署名を追加する:

  ```
  ---
  🤖 *この Issue は Claude によって自動作成されました*
  ```

  ```bash
  gh issue create \
    --title "{prefix}{タイトル}" \
    --body "{組み立てた本文}

  ---
  🤖 *この Issue は Claude によって自動作成されました*" \
    --label "{label}"
  ```

  作成後、Issue の URL をユーザーに伝える。

  ---

  ## Step 6: 継続確認

  AskUserQuestion で聞く:
  「他にも Issue を作りますか？」

  - Yes → Step 1 に戻る
  - No → 終了
